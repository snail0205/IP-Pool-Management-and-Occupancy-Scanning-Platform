### 一、项目目标与范围

项目名称: IP池管理与占用扫描平台
目标: 对指定网段进行资产可占用可视化管理,实现IP池初始化，定时扫描、状态判断、异常警告与历史追踪。
使用角色:

 - 管理员: 配置网段、触发扫描、维护备案信息、处理冲突
 - 普通查看者(可选): 仅查看状态矩阵和统计信息



### 二、技术栈选型

后端: Node.js + Express.js

数据库: MySQL 8

实时通信: Socket.io

扫描核心：

- ping: 基础存活探测 

- node-libnmap: 端口与主机信息深度探测

前端: Vue3

#### 补充:

​	任务调度: node-cron(定时扫描)
​	日志: winston/pino
​	参数校验: joi/zod
​	CIDR/IP工具: ipaddr.js 或 ip-address
​	注意点: node-libnmap依赖系统安装nmap,部署文档明确环境前置条件

### 三、核心功能需求(优化版)

#### 3.1 IP池定义与自动计算

##### 3.1.1 功能描述

​	管理输入CIDR(10.0.0.0/24), 系统自动生成网段内IP列表并初始化状态。

##### 3.1.2 业务规则

 - 支持CIDR格式校验,不合法拒绝保存

 - 支持网段大小限制(最大/16,防止超大网段拖垮扫描)

 - 关于地址范围需明确策略(二选一):

   1.包含网络地址和广播地址
   2.仅主机地址(/24常见为254)

-  首次导入默认状态"未知/未扫描"

-  允许重新导入时选择"覆盖/增量合并"

##### 3.1.3 数据落库

初始化写入ip_status

#### 3.2 异步扫描引擎

##### 3.2.1 功能描述

系统按并发策略执行Ping+Nmap扫描,解析结果并更新数据库,支持实时推送进度。

##### 3.2.2 扫描流程

1.创建扫描任务(task_id)

2.按并发批量取IP(Promise.all/async.eachLimit)

3.先执行Ping(快速筛选)

4.对存活IP执行Nmap深扫(端口/MAC/OS 可选)

5.解析stdout(正则提取TTL、MAC、端口)

6.状态判定(与备案对比)

7.写入结果 + 通过Socket.io 推送前端

8.任务结束,产出统计(在线数，异常数，耗时)

##### 3.2.3 并发性与性能需求

- 并发数可配置(默认20~50)
- 扫描超时可配置(如ping 1~2s, nmap 3~10s)
- 同一网段同一时刻仅允许一个进行中任务(防重复)
- 支持任务中断与重试

##### 3.2.4 异常处理

- 命令执行失败: 记录错误码与 stderr
- 超时: 标记timeout,不中断整体任务
- 解析失败: 保留原始输出供神级
- 数据库写失败: 重试+错误日志



#### 3.3 IP状态矩阵展示

##### 3.3.1 展示逻辑

前端Vue3获取IP列表后,使用v-for网格化渲染(256格或动态数)

##### 3.3.2 状态定义(建议升级为4态)

你现在是3态,建议增加"未知态"更严谨:

-  -1(Blue/Gray): 未知(未扫描或扫描中)
-  0 (Gray):未占用(不可达,且无备案占用)
- 1 (Green): 正常占用(可达,且与备案一致)
- 2 (Red):冲突/非法占用(可达但备案不一致/MAC变更)

##### 3.3.3 交互需求

-  点击格子查看详情: IP、MAC、TTL、端口、最后是扫描时间
- 支持筛选 (全部/在线/冲突/未占用)
- 支持按关键字搜索IP或MAC
- 实时更新进度与在线统计

### 七、接口设计（REST API）

#### 7.1 统一约定

- 协议：HTTP/HTTPS（开发环境用 HTTP，生产建议用 HTTPS）
- 数据格式：`application/json; charset=utf-8`
- 字符编码：UTF-8
- 鉴权方式：Bearer Token
- 时间格式：`YYYY-MM-DD HH:mm:ss`
- 状态码约定：
  - 200 请求成功
  - 201 创建成功
  - 400 参数错误
  - 401 未登录/Token失效
  - 403 无权限
  - 404 资源不存在
  - 409 资源冲突（如重复启动同网段任务）
  - 500 服务器内部错误

#### 7.2 统一响应结构

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "20260215-xxxx",
  "timestamp": "2026-02-15 20:00:00"
}
```

- `code = 0` 表示业务成功，非 0 表示业务失败。
- 建议所有接口统一返回该结构，便于前端统一拦截处理。

#### 7.3 IP池管理接口

##### 7.3.1 创建网段池并初始化IP

- URL：`POST /api/pools`
- 权限：管理员
- 请求参数：

```json
{
  "name": "办公网A区",
  "cidr": "10.0.0.0/24",
  "includeNetworkAndBroadcast": false,
  "initMode": "overwrite"
}
```

- 参数说明：
  - `name`：池名称
  - `cidr`：网段
  - `includeNetworkAndBroadcast`：是否包含网络地址和广播地址
  - `initMode`：`overwrite` 覆盖初始化 / `merge` 增量合并
- 成功响应：

```json
{
  "code": 0,
  "message": "pool created",
  "data": {
    "poolId": 1,
    "cidr": "10.0.0.0/24",
    "totalIpCount": 254
  }
}
```

##### 7.3.2 获取网段池列表

- URL：`GET /api/pools`
- 权限：管理员/只读
- 查询参数：`page`、`pageSize`、`keyword`
- 响应重点字段：`poolId`、`name`、`cidr`、`totalIpCount`、`lastScanTime`

##### 7.3.3 获取池内IP列表与状态

- URL：`GET /api/pools/:id/ips`
- 权限：管理员/只读
- 查询参数（可选）：
  - `status`：`-1|0|1|2`
  - `keyword`：IP/MAC/设备名
  - `page`、`pageSize`
- 成功响应示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total": 254,
    "list": [
      {
        "ip": "10.0.0.12",
        "status": 1,
        "isAlive": true,
        "mac": "AA:BB:CC:DD:EE:FF",
        "ttl": 64,
        "openPorts": [22, 80],
        "lastScanTime": "2026-02-15 19:58:20"
      }
    ]
  }
}
```

#### 7.4 扫描任务接口

##### 7.4.1 启动扫描任务

- URL：`POST /api/scan/:poolId/start`
- 权限：管理员
- 请求参数：

```json
{
  "concurrency": 30,
  "pingTimeoutMs": 1500,
  "nmapTimeoutMs": 6000,
  "deepScan": true
}
```

- 业务规则：同一 `poolId` 同时仅允许一个 `running` 任务。
- 成功响应：

```json
{
  "code": 0,
  "message": "scan started",
  "data": {
    "taskId": 10001,
    "poolId": 1,
    "status": "running"
  }
}
```

##### 7.4.2 查询任务进度

- URL：`GET /api/scan/tasks/:taskId`
- 权限：管理员/只读
- 成功响应重点字段：
  - `status`：`pending/running/success/failed/cancelled`
  - `progress`：`0~100`
  - `processedCount`
  - `onlineCount`
  - `conflictCount`
  - `startedAt`、`endedAt`
  - `errorMsg`

##### 7.4.3 取消扫描任务（可选）

- URL：`POST /api/scan/tasks/:taskId/cancel`
- 权限：管理员
- 说明：仅 `running` 任务允许取消。

##### 7.4.4 任务历史列表

- URL：`GET /api/scan/tasks/history`
- 权限：管理员/只读
- 查询参数：`poolId`、`status`、`startTime`、`endTime`、`page`、`pageSize`

#### 7.5 备案管理接口

##### 7.5.1 新增备案

- URL：`POST /api/registry`
- 权限：管理员
- 请求参数：

```json
{
  "poolId": 1,
  "ip": "10.0.0.12",
  "expectedMac": "AA:BB:CC:DD:EE:FF",
  "deviceName": "实验室打印机01",
  "owner": "张三",
  "department": "网络中心",
  "remark": "固定资产"
}
```

##### 7.5.2 更新备案

- URL：`PUT /api/registry/:id`
- 权限：管理员
- 说明：记录变更审计（操作人、时间、变更前后值）。

##### 7.5.3 删除备案

- URL：`DELETE /api/registry/:id`
- 权限：管理员

### 八、Socket事件设计

#### 8.1 连接与鉴权

- 命名空间建议：`/scan`
- 客户端连接时携带 Token
- 服务端校验失败返回 `connect_error`
- 建议按网段池加入房间：`room:pool:{poolId}`，减少无关广播

#### 8.2 事件清单（服务端 -> 客户端）

##### 8.2.1 `scan:started`

- 触发时机：任务创建并开始执行
- payload：

```json
{
  "taskId": 10001,
  "poolId": 1,
  "status": "running",
  "startedAt": "2026-02-15 20:00:00"
}
```

##### 8.2.2 `scan:progress`

- 触发时机：每处理一批IP或固定时间间隔推送
- payload：

```json
{
  "taskId": 10001,
  "poolId": 1,
  "progress": 45,
  "processedCount": 114,
  "totalCount": 254,
  "onlineCount": 72,
  "conflictCount": 3,
  "timestamp": "2026-02-15 20:01:10"
}
```

##### 8.2.3 `scan:ip-updated`

- 触发时机：单个IP状态发生变化后
- payload：

```json
{
  "taskId": 10001,
  "poolId": 1,
  "ip": "10.0.0.12",
  "status": 2,
  "isAlive": true,
  "mac": "AA:BB:CC:DD:EE:00",
  "ttl": 64,
  "openPorts": [22, 80],
  "reason": "mac_mismatch",
  "lastScanTime": "2026-02-15 20:01:12"
}
```

##### 8.2.4 `scan:finished`

- 触发时机：任务结束（成功/失败/取消）
- payload：

```json
{
  "taskId": 10001,
  "poolId": 1,
  "status": "success",
  "durationMs": 183000,
  "summary": {
    "totalCount": 254,
    "onlineCount": 160,
    "freeCount": 90,
    "conflictCount": 4,
    "unknownCount": 0
  },
  "endedAt": "2026-02-15 20:03:03"
}
```

##### 8.2.5 `scan:error`

- 触发时机：任务执行异常（不中断或中断都可推送）
- payload：

```json
{
  "taskId": 10001,
  "poolId": 1,
  "level": "error",
  "errorCode": "NMAP_TIMEOUT",
  "message": "nmap timeout on 10.0.0.45",
  "timestamp": "2026-02-15 20:01:50"
}
```

#### 8.3 事件清单（客户端 -> 服务端，可选）

- `scan:subscribe`：订阅指定 `poolId` 的实时事件
- `scan:unsubscribe`：取消订阅
- `scan:ping`：心跳探活（也可直接用 Socket.io 内建心跳）

#### 8.4 可靠性与性能策略

- `scan:progress` 建议节流推送（如每 500ms~1000ms 一次）
- 对 `scan:ip-updated` 可按批次合并，避免高频事件风暴
- 客户端断线重连后，通过 `GET /api/scan/tasks/:taskId` 补拉状态
- 服务端记录最后一次任务快照，避免前端错过关键结果

### 十、新增功能需求（地址分配、在线扫描、冲突检测、权限、报表）

#### 10.1 地址分配管理

- 支持在网段池内进行IP地址分配、释放、批量导入与导出。
- 分配记录至少包含：`IP`、`设备名称`、`责任人`、`部门`、`分配时间`、`状态`。
- 新建网段池时自动完成地址初始化，写入 `ip_scan_result` 默认状态为 `-1(未知)`。
- 支持按条件查询（IP、责任人、部门、是否已分配）。

#### 10.2 在线占用扫描

- 支持管理员手动触发扫描任务，后端按并发参数执行在线探测（Day3先完成 Ping 探测）。
- 扫描过程中持续更新任务进度：`totalCount`、`processedCount`、`progress`。
- 扫描结果写入 `ip_scan_result`，记录 `is_alive`、`ttl`、`status_code`、`last_scan_time`。

#### 10.3 冲突检测

- 冲突判定规则（Day3）：
  - IP在线且无备案记录 => `status_code=2`（疑似非法占用）
  - IP在线且存在备案记录 => `status_code=1`（正常占用）
  - IP离线 => `status_code=0`（未占用）
- 后续增强（Day4+）：引入 MAC 比对后增加 `mac_mismatch` 细分冲突类型。

#### 10.4 权限与安全设计

- 角色划分：管理员（可写）与只读用户（只查）。
- 接口权限策略：
  - 写操作（创建池、启动扫描、更新备案）仅管理员可用。
  - 查操作（池列表、IP状态、任务进度）管理员和只读用户可用。
- Day3阶段先保留鉴权中间件入口，Day4阶段接入真实 Token 校验。

#### 10.5 报表功能

- 支持按网段池和时间范围生成统计报表。
- 报表核心指标：在线数、未占用数、冲突数、占用率、扫描耗时。
- 支持 CSV/Excel 导出（后续迭代实现）。
- Day3阶段先保证报表数据源字段齐全（任务表 + 扫描结果表）。
